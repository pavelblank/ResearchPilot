# Project Structure — The 4 Standard Folders

Every project in `01-PROJECTS/` has these 4 subfolders. They are created automatically when you make a new project through the UI (Settings → Projects → + Create).

This `00-TEMPLATE/` folder is a **blueprint only** — do not upload papers here. Create your own project (e.g., `P1-MY-RESEARCH`) and use these folder names.

| Folder | Purpose | What goes here |
|---|---|---|
| **01-LIBRARY** | Source materials | PDFs, DOCX, PPTX, HTML, TXT, MD — anything you upload. Auto-converted to markdown on upload. |
| **02-EXTRACTIONS** | AI analyses | The 12-point elite extraction of every paper (APA, DOI, Quartile, Method, Framework, Limitations, etc.). Auto-generated when a paper is processed. |
| **03-MANUSCRIPTS** | Your writing | Drafts, chapters, literature reviews, abstracts. Editable markdown. |
| **99-META** | Project notes | Project-level context: research questions, scope, methodology decisions, progress notes. Helps the AI understand the project when you chat. |

## Quick start

1. Open ResearchPilot in your browser.
2. Go to **⚙️ Settings → Projects**.
3. Click **+ Create** and name your project (e.g., `P1-CLIMATE-POLICY`).
4. The 4 folders above are created automatically.
5. Drag your PDFs into `01-LIBRARY/01-LIBRARY/` through the web UI's **Upload** tab.

## Naming convention

For best results, use a sortable prefix like:
- `P1-`, `P2-`, `P3-` for sequential projects
- `A-`, `B-` for parallel projects
- `00-`, `01-` for sub-projects

This keeps the project list sorted and easy to navigate.
