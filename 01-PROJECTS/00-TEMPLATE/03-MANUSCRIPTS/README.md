# 03-MANUSCRIPTS

This folder holds your **writing** — drafts, chapters, literature reviews, abstracts, and any other markdown documents you create as part of the project.

## How to use

- Create new `.md` files directly in this folder using any text editor.
- Reference them in the web UI's **Manuscripts** tab.
- The chat assistant can read these files (and any file in your ResearchPilot system) using the built-in `read_file` tool.

## Suggested structure

```
03-MANUSCRIPTS/
├── chapter-1-introduction.md
├── chapter-2-literature-review.md
├── chapter-3-methodology.md
├── chapter-4-findings.md
├── chapter-5-discussion.md
├── chapter-6-conclusion.md
└── drafts/
    ├── draft-2026-06-01.md
    └── draft-2026-06-04.md
```

## Why markdown?

- Plain text survives any tool change (Word, LaTeX, Notion, Obsidian).
- Version control with git works perfectly.
- AI engines can read and quote it directly.
- You can convert to PDF, DOCX, or HTML using Pandoc, Obsidian, or VS Code.
