## 🔬 Section-by-Section Details

### 1. Introduction
- **Shift**: Moving from the "Human-as-a-Weak-Link" paradigm to the "Human-as-a-Security-Partner" paradigm.
- **Problem**: Security tools that are technically perfect but unusable in real-world contexts.

### 2. Systematic Literature Review
- **Scope**: Multi-decade review of HCI (Human-Computer Interaction) and Security papers.
- **Key Trend**: The rise of "Invisible Security" and "Nudging" in design.

### 3. The Human Factors Taxonomy
- **Dimension 1: Cognitive Load**: Measuring the mental effort required to use security features (e.g., MFA complexity).
- **Dimension 2: Mental Model Alignment**: Ensuring the system's logic matches the user's intuitive understanding of security.
- **Dimension 3: User Agency**: Providing users with clear feedback and control over security actions.

### 4. Usability-Compliance Linkage
- **Mechanism**: High Usability -> Low Cognitive Load -> High Adherence (No need for workarounds).
- **Relevance for Project 1**: This provides the **"Technical-Cultural Alignment"** link.

### 5. Conclusion & Recommendations
- **Actionable Insight**: HEI security architects must involve academic staff in the **Design Phase** of security policies to ensure cultural readiness.
